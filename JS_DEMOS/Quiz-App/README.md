# DevQuiz // Interactive Frontend Engineering Assessment

DevQuiz is a professional, responsive interactive quiz platform engineered cleanly using standard web components: **Semantic HTML5**, **Modern CSS3 (Glassmorphism architecture)**, and **Event-driven Vanilla JavaScript (ES6+)**. 

This application demonstrates optimal state management paradigms and clean separation of structural presentation, styling templates, and behavioral logic engines.

---

## 🚀 Features

- **Dynamic DOM Rendering Pipeline:** Questions, choice indices, and UI interaction tracking nodes are bound dynamically at run-time.
- **Robust State Validation Layers:** Locks operational interfaces upon selection arrays to completely eliminate double-clicking bugs.
- **Visual Feedback Engines:** Correct and incorrect options transitions fire instant, clear colors directly into target components.
- **Heuristics Assessment HUD:** Real-time progress bar recalculations track scores, indices, and completion percentage values instantly.
- **Conditional Tiering Strategy Engine:** Formulates varied, responsive performance icons and evaluation statements based on score milestones.

---

## 🛠️ Technologies Used

- **Structure Layer:** HTML5 Semantic Engine
- **Styling Matrix:** CSS3 Custom Variables, Flexbox Component Arrays, Custom Animation Keyframes
- **Logical Behavior Suite:** Vanilla ECMAScript 6+ standard patterns
- **Typography:** Plus Jakarta Sans engine via Google Font APIs

---

## 💡 Learning Outcomes & JS Concepts Practiced

1. **Structured Data Schemas (Objects within Arrays):** Organizing complex quiz matrices mapping internal keys (`question`, `options`, `correct`) cleanly.
2. **Iterative Component Mapping (`forEach`):** Traversing alternative responses arrays to output consistent, standard UI components.
3. **Advanced Event Handling listeners:** Binding isolated event loops to buttons securely to govern execution variables without cross-pollution.
4. **DOM Token Class Controls (`classList`):** Toggling structural layouts variables (`.active`, `.hidden`, `.correct`, `.incorrect`) efficiently.
5. **State Inversion Paradigms:** Tracking global parameters indices smoothly to govern screen rendering logic.

---

## 📦 Installation Instructions

1. Clone or download these codebase components into your desired workspace directory.
2. Verify all three fundamental operational assets (`index.html`, `style.css`, and `script.js`) occupy the same layout file folder.
3. Launch `index.html` within your preferred target web browser interface, or deploy it through the VS Code Live Server helper extension.
