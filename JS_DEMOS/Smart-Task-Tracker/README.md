# TaskFlow // Smart Task Tracker

TaskFlow is a production-grade, highly responsive task tracking web application built entirely using semantic standard structural layers (**HTML5**), glassmorphic layout definitions (**CSS3**), and decoupled event-driven behavior engines (**Vanilla JavaScript**). 

This project goes beyond basic programming templates to model a modular UI workflow designed with portfolio presentation requirements in mind.

---

## 🚀 Features

- **Asynchronous CRUD Layer:** Add, edit, track, or wipe operations seamlessly with instant reactivity.
- **Dynamic Metric Engine:** Computes complete vs open workloads to recalculate a real-time responsive dashboard progress visualizer.
- **Granular Priority Scopes:** Tag jobs across low, medium, and high importance variables tied cleanly to context-specific theme tags.
- **State Partition Routing:** Swap dynamic list perspectives instantaneously matching `All`, `Active`, or `Completed` item parameter flags.
- **System Memory Anchor:** Keeps runtime modifications structural, parsing changes seamlessly into the browser's persistent LocalStorage cache.
- **Clean Fallback Interventions:** Renders structural, polished placeholder layouts dynamically whenever application queues rest empty.
- **Toast Notifications:** Displays crisp, context-relevant non-blocking feedback popups on system updates.

---

## 🛠️ Technologies Used

- **Markup Language:** HTML5 (Semantic Structure)
- **Styling Architecture:** CSS3 (Custom Variables, Flexbox Grid Systems, Keyframes, Glassmorphism mechanics)
- **Programming Engine:** Vanilla ECMAScript 6+ 
- **Typography Matrix:** Plus Jakarta Sans Engine via Google Web Fonts

---

## 🧠 JavaScript Concepts Practiced

1. **State Machine Principles:** Consolidating global arrays as singular points of reference to feed runtime viewports cleanly (`render()`).
2. **Event Delegation:** Mitigates memory bloating by attaching generic tactical interaction interceptors to structural parents (`#task-list`) instead of generating per-node bindings.
3. **Array Mutation Pipelines:** Mastering structural mutations with `map()`, `filter()`, and `unshift()` array functions.
4. **DOM Fragment Batching:** Assembles elements inside offscreen document frames before committing insertions to reduce page-repaint layouts.
5. **Security Protocols (XSS Mitigation):** Sanitizes inputs against cross-site scripting risks using dedicated character entity conversion wrappers.

## 📦 How to Run

1. Clone or download these repository contents into your environment.
2. Ensure the three primary files (`index.html`, `style.css`, and `script.js`) exist in the same root folder.
3. Open the `index.html` file using your web browser of choice, or run it via an extension like Live Server in VS Code.
